# Gold Advisor Midnight — Data File Encoding Howto

This document covers the XOR + custom base64 encoding system used to protect
`StratsGenerated.lua` and `WorkbookGenerated.lua`.

---

## Overview

The addon ships in two build modes:

| Build | Data files | Use case |
|-------|-----------|---------|
| Dev / open-source | Plain `StratsGenerated.lua`, `WorkbookGenerated.lua` | Development, GitHub |
| Protected release | Encoded `StratsEncoded.lua`, `WorkbookEncoded.lua` | Paid/distributed release |

The addon code (all `.lua` except data files) stays plain text — Blizzard can
review it. Only the content data is protected.

Without data files the addon loads normally, scans normally, and opens the UI —
it simply shows no strategies. This is already handled gracefully in the code:
- `Importer.Init()` checks for nil tables and logs a warning
- `Constants.lua` `ProfileDefault()` returns hardcoded fallbacks if workbook is nil

---

## Encoding Format

**Steps (encode direction):**
1. Read the plain `.lua` file as raw bytes (binary mode — preserves CRLF endings)
2. XOR every byte with a repeating key: `MidnightGold2026`
3. Standard base64-encode the XOR'd bytes
4. Translate the standard base64 output character-by-character using a custom
   shuffled 64-char alphabet (see below) — naive base64 decoders produce garbage
5. Write the result as a Lua string literal:

```lua
-- Encoded data file. Do not edit manually.
-- Generated by tools/encode_data.py
GAM_STRATS_ENCODED = "<encoded string here>"
```

**XOR key:** `MidnightGold2026`

**Custom base64 alphabet (64 chars, NOT standard order):**
```
QRSTUVWXYZabcdefvutsrqponmlkjihgwxyz0123456789+/ABCDEFGHIJKLMNOP
```
Standard alphabet for reference:
```
ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789+/
```

## Running the Tools

### Encode (plain → protected)

```bash
cd /path/to/GoldAdvisorAddon
python tools/encode_data.py
```

Reads from `source/GoldAdvisorMidnight/Data/`:
- `StratsGenerated.lua`   → `StratsEncoded.lua`
- `WorkbookGenerated.lua` → `WorkbookEncoded.lua`

Override key (if you change it):
```bash
python tools/encode_data.py --key YourNewKey
```

### Decode (protected → plain, for verification)

```bash
# Print to stdout
python tools/decode_data.py source/GoldAdvisorMidnight/Data/StratsEncoded.lua

# Write to file
python tools/decode_data.py source/GoldAdvisorMidnight/Data/StratsEncoded.lua --out decoded_strats.lua

# Quick sanity check — peek at first 20 lines
python tools/decode_data.py source/GoldAdvisorMidnight/Data/StratsEncoded.lua | head -20
```

### Round-trip verification

```bash
python tools/encode_data.py
python tools/decode_data.py source/GoldAdvisorMidnight/Data/StratsEncoded.lua --out /tmp/strats_rt.lua
diff source/GoldAdvisorMidnight/Data/StratsGenerated.lua /tmp/strats_rt.lua && echo "IDENTICAL"
```

## Future: Lua Decoder (Protected Release Build)

When ready to ship encoded builds, add the following fragments to **existing files only**
(no new `.lua` files needed). All fragments look like plausible addon internals.

### Key fragments — add to existing files

**`Core.lua`** (bottom of file, near other module-level setup):
```lua
GAM._vt = "Midnight"   -- internal version tag
```

**`Log.lua`** (bottom, near log-level setup):
```lua
GAM._ls = "Gold"       -- log-sink identifier
```

**`Minimap.lua`** (bottom, near icon/texture constants):
```lua
GAM._ms = "2026"       -- minimap stamp constant
```

Assembled key: `GAM._vt .. GAM._ls .. GAM._ms` → `"MidnightGold2026"`

### Base64 alphabet halves — add to existing files

**`Locale.lua`** (bottom, near locale fallback helpers):
```lua
GAM._ea = "QRSTUVWXYZabcdefvutsrqponmlkjihg"   -- first 32 chars of encoding alphabet
```

**`Pricing.lua`** (bottom, near price-format constants):
```lua
GAM._eb = "wxyz0123456789+/ABCDEFGHIJKLMNOP"   -- last 32 chars of encoding alphabet
```

Assembled alphabet: `GAM._ea .. GAM._eb`

### Decode logic — add to `Importer.lua`

Add these private functions near the top of `Importer.lua`, before `Importer.Init`:

```lua
-- ── Encoded data loader (do not rename or move) ────────────────────────────

local function _b64dec(s)
    local alpha = (GAM._ea or "") .. (GAM._eb or "")
    if #alpha ~= 64 then return nil end
    local lookup = {}
    for i = 1, 64 do
        lookup[alpha:sub(i, i)] = i - 1
    end
    -- strip whitespace / newlines
    s = s:gsub("[^" .. alpha .. "=]", "")
    local out = {}
    local i = 1
    while i <= #s do
        local c1 = lookup[s:sub(i,i)]     or 0
        local c2 = lookup[s:sub(i+1,i+1)] or 0
        local c3 = lookup[s:sub(i+2,i+2)] or 0
        local c4 = lookup[s:sub(i+3,i+3)] or 0
        local n = c1 * 262144 + c2 * 4096 + c3 * 64 + c4
        local b1 = math.floor(n / 65536)
        local b2 = math.floor((n % 65536) / 256)
        local b3 = n % 256
        out[#out+1] = string.char(b1)
        if s:sub(i+2,i+2) ~= "=" then out[#out+1] = string.char(b2) end
        if s:sub(i+3,i+3) ~= "=" then out[#out+1] = string.char(b3) end
        i = i + 4
    end
    return table.concat(out)
end

local function _xordec(data, key)
    local klen = #key
    local out = {}
    for i = 1, #data do
        local kb = key:byte(((i - 1) % klen) + 1)
        out[i] = string.char(data:byte(i) ~ kb)   -- Lua 5.4 bitwise XOR
        -- For WoW (Lua 5.1 / LuaJIT): use bit.bxor(data:byte(i), kb)
    end
    return table.concat(out)
end

local function _loadEncoded(encoded)
    if not encoded or encoded == "" then return end
    local key = (GAM._vt or "") .. (GAM._ls or "") .. (GAM._ms or "")
    local decoded = _b64dec(encoded)
    if not decoded then
        GAM.Log.Warn("GAM", "Failed to base64-decode data blob")
        return
    end
    local src = _xordec(decoded, key)
    local fn, err = load(src)
    if fn then
        fn()
    else
        GAM.Log.Warn("GAM", "Failed to load encoded data: " .. tostring(err))
    end
end
```

> **WoW Lua note:** WoW uses Lua 5.1 / LuaJIT. Replace the `~` bitwise XOR
> operator with `bit.bxor(a, b)` — the `bit` library is available in WoW.

Then in `Importer.Init()`, replace the plain data file calls with:
```lua
_loadEncoded(GAM_STRATS_ENCODED)
_loadEncoded(GAM_WORKBOOK_ENCODED)
```

### TOC changes for protected build

In `GoldAdvisorMidnight.toc`, swap the plain data file lines:
```
# Replace these two lines:
Data\StratsGenerated.lua
Data\WorkbookGenerated.lua

# With:
Data\StratsEncoded.lua
Data\WorkbookEncoded.lua
```

The open-source / dev build keeps the plain `.lua` files in the TOC. The
protected release build uses the encoded ones.

## Release Build Workflow (future)

When `Package_Addon.command` is updated to support protected builds, the flow is:

1. Edit strategies / workbook data (plain `.lua` files as normal)
2. Run `python tools/encode_data.py` to produce encoded blobs
3. Build the zip using the TOC that references `*Encoded.lua` files
4. The plain `*Generated.lua` files are **not included** in the protected zip
5. The `tools/` directory is **not included** in the zip

## Security Notes

- This is **obfuscation**, not strong encryption. Anyone with enough determination
  and Lua knowledge can reverse it by reading `Importer.lua`.
- The goal is to raise the barrier high enough that casual copy-paste theft is
  prevented and that it requires meaningful reverse-engineering effort.
- The custom base64 alphabet means any naive base64 decoder will produce garbage,
  eliminating the fastest attack vector.
- Key fragments scattered across 5 files means the attacker must read and
  understand multiple files, not just `Importer.lua`.
- The addon code itself (all non-data `.lua` files) is always open/reviewable by
  Blizzard as required by their addon policy.
